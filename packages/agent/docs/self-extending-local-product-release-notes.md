# Worker-First Local Product Notes

These notes summarize the completed self-extending productization campaign from
the current worker-first product vocabulary. This is product history, not a
release, source-control publishing, notarization, or rollout checklist.

## Product Shape

Tron is productized as worker-led autonomous work on this Mac. The main flow is
one orchestrator plus workers: a user asks for work, Tron runs
run-unless-blocked, delegates to Workers and Worker Packs, shows progress in
the Work dashboard, and keeps raw proof in Audit Details.

The main vocabulary is Work, Workers, Worker Packs, Autonomy, Guardrails, and
Audit. The app renders server-owned Generated Controls, trust labels, model
routing, and action summaries rather than owning product truth locally.

## User-Visible Changes

- Chat can start work while the Work dashboard shows autonomous progress.
- Worker detail sheets give a durable place to revisit locally created workers
  and Worker Packs.
- Worker Pack rows use product language while Audit Details keeps exact
  package, source, activation, trust, conformance, generated-control, and
  invocation details.
- Generated Controls let new worker actions appear without app updates when
  they fit the stable native catalog.
- Model presets are Local when possible, Balanced, and Deep. Selected routes
  and fallbacks are visible after the server decides.
