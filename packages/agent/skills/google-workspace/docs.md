# Google Docs

## Helper commands

### Append text
```bash
gws docs +write --document DOC_ID --text 'Hello, world!'
```

Inserts plain text at the end of the document. For rich formatting, use the raw batch update API.

## Raw API

### Documents

```bash
# Create document
gws docs documents create --json '{"title":"Meeting Notes — March 2026"}'

# Get document (full content)
gws docs documents get --params '{"documentId":"DOC_ID"}'

# Batch update (insert, delete, format)
gws docs documents batchUpdate --params '{"documentId":"DOC_ID"}' --json '{
  "requests": [
    {"insertText": {"location": {"index": 1}, "text": "# Report Title\n\nGenerated by Tron.\n\n"}}
  ]
}'
```

### Common batch update requests

**Insert text at a position:**
```json
{"insertText": {"location": {"index": 1}, "text": "New text"}}
```

**Delete text:**
```json
{"deleteContentRange": {"range": {"startIndex": 1, "endIndex": 50}}}
```

**Replace all occurrences:**
```json
{"replaceAllText": {"containsText": {"text": "OLD", "matchCase": true}, "replaceText": "NEW"}}
```

**Insert page break:**
```json
{"insertPageBreak": {"location": {"index": 1}}}
```

**Insert table:**
```json
{"insertTable": {"rows": 3, "columns": 2, "location": {"index": 1}}}
```

## Workflow: create a report

```bash
# 1. Create the doc
gws docs documents create --json '{"title":"Weekly Report — 2026-W11"}'
# Note the documentId from the response

# 2. Write content
gws docs +write --document DOC_ID --text 'Summary of the week...'

# 3. Optionally upload to a Drive folder
gws drive files update --params '{"fileId":"DOC_ID","addParents":"FOLDER_ID"}'

# 4. Share it
gws drive permissions create --params '{"fileId":"DOC_ID"}' --json '{"role":"reader","type":"user","emailAddress":"moose@example.com"}'
```

## Export

Google Docs can be exported via the Drive API:
```bash
# As PDF
gws drive files export --params '{"fileId":"DOC_ID","mimeType":"application/pdf"}' --output ./report.pdf

# As DOCX
gws drive files export --params '{"fileId":"DOC_ID","mimeType":"application/vnd.openxmlformats-officedocument.wordprocessingml.document"}' --output ./report.docx

# As plain text
gws drive files export --params '{"fileId":"DOC_ID","mimeType":"text/plain"}' --output ./report.txt
```

## Gotchas

- Document content uses 1-based indexing. Index 1 is the start of the body.
- The `+write` helper only appends plain text. Use `batchUpdate` for positioning, formatting, tables, or images.
- `batchUpdate` requests are applied in order. If inserting text, subsequent indices shift — work backwards or adjust indices.
- Creating a doc returns both a `documentId` and a `revisionId`. The `documentId` is also the Drive file ID.
- Reading a full document returns the entire structural representation (paragraphs, elements, styles). For quick text extraction, export as plain text instead.
